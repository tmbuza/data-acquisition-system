End-to-End Case Study

Published

Jun 2026

  • ID: DAS-009
  • Type: Case Study
  • Audience: Omics Data Scientists, Bioinformaticians, and Research Teams
  • Theme: Applying the CDI Data Acquisition System from Study Discovery to Reference Dataset Assembly

The previous chapters introduced the individual components of the CDI Data Acquisition System. This chapter demonstrates how these components work together as a complete end-to-end workflow.

The objective is to illustrate how publicly available sequencing studies can be transformed into a validated and reproducible reference dataset.

Rather than focusing on a specific organization or project, this case study presents a generalized workflow that can be adapted to a wide range of omics data acquisition scenarios.

Case Study Objective

Objective:

Build a healthy reference gut microbiome dataset from publicly available sequencing studies.

The goal is to demonstrate how the CDI Data Acquisition System can transform public repository data into a curated analytical resource.

Expected outputs include:

  • Metadata assets
  • Download manifests
  • Sequencing files
  • Validation reports
  • Cloud storage assets
  • Reference dataset components

End-to-End System Workflow

Code 
flowchart TD

A[Study Discovery]
--> B[Metadata Acquisition]

B --> C[Data Download]

C --> D[Data Validation]

D --> E[Cloud Storage and Transfer]

E --> F[Reference Dataset Assembly]

F --> G[Reference Dataset Release]

Each stage contributes a reproducible component to the final dataset.

Step 1: Study Discovery

The process begins by identifying candidate studies relevant to the dataset objective.

Example workflow:

Literature Review
        ↓
Public Repositories
        ↓
Candidate Studies

Example candidate studies:

PRJNA477349
PRJNA802976
PRJNA322554

Key activities:

  • Review scientific literature
  • Identify relevant studies
  • Evaluate study eligibility
  • Record accession information

Output:

Candidate Studies

Step 2: Metadata Acquisition

Once studies have been identified, metadata are retrieved from public repositories.

Example workflow:

BioProject
        ↓
Run Metadata
        ↓
ENA Metadata
        ↓
Sample Inventory

Example outputs:

data/metadata/
├── runinfo-prjna477349.csv
├── ena-prjna477349.tsv
└── srr-accessions.txt

Metadata acquisition provides the foundation for downstream filtering and sample selection.

Output:

Metadata Assets

Step 3: Metadata Review and Sample Selection

Metadata are reviewed against predefined inclusion and exclusion criteria.

Example inclusion criteria:

  • Human samples
  • Healthy individuals
  • Stool specimens
  • Publicly available sequencing data

Example exclusion criteria:

  • Disease status unknown
  • Missing metadata
  • Incomplete records
  • Duplicate samples

Workflow:

Metadata Assets
        ↓
Inclusion Criteria
        ↓
Exclusion Criteria
        ↓
Eligible Samples

Output:

Download Manifest

Step 4: Data Download

Validated metadata are used to retrieve sequencing data.

Example workflows:

ENA Retrieval

bash scripts/bash/05b-download-ena-fastq.sh

NCBI Retrieval

bash scripts/bash/05c-download-ncbi-sra.sh

Example outputs:

data/raw/
├── ena/
└── ncbi/

Workflow:

Download Manifest
        ↓
Repository Retrieval
        ↓
FASTQ Files

Output:

Sequencing Data

Step 5: Download Verification

Downloaded files are verified against repository expectations.

Verification activities:

  • File count verification
  • Download completeness
  • Inventory generation
  • Repository reconciliation

Example:

bash scripts/bash/05d-verify-downloads.sh

Inventory generation:

bash scripts/bash/05e-build-fastq-inventory.sh

Output:

FASTQ Inventory

Step 6: Data Validation

Downloaded files undergo systematic validation before being used for downstream analysis.

Example:

bash scripts/bash/06-validate-download.sh

Validation workflow:

Repository Validation
        ↓
File Validation
        ↓
Compression Validation
        ↓
Paired-End Validation
        ↓
FASTQ Validation

Example output:

check   status  value
fastq_file_count    PASS    6
compression_validation  PASS    0
paired_end_validation   PASS    3
fastq_validation    PASS

Output:

Validated Dataset

Step 7: Cloud Storage and Transfer

Validated datasets are transferred to durable storage.

Example:

bash scripts/bash/07-transfer-to-cloud.sh \
  data \
  my-reference-datasets

Example storage structure:

s3://my-reference-datasets/
├── metadata/
├── manifests/
├── inventory/
├── raw/
└── validation/

Workflow:

Validated Dataset
        ↓
Cloud Transfer
        ↓
Cloud Storage

Output:

Cloud Repository

Step 8: Reference Dataset Assembly

Validated data and metadata are integrated into a coherent reference dataset.

Workflow:

Validated Samples
        ↓
Metadata Harmonization
        ↓
Dataset Integration
        ↓
Reference Dataset

Example structure:

reference-dataset/
├── metadata/
├── harmonized-metadata/
├── documentation/
└── dataset-release/

Key activities:

  • Metadata harmonization
  • Sample integration
  • Provenance preservation
  • Documentation

Output:

Reference Dataset

Reference Dataset

Reproducibility Considerations

A reproducible reference dataset should preserve:

  • Source repositories
  • Study accessions
  • Sample identifiers
  • Download manifests
  • Validation reports
  • Storage locations
  • Dataset documentation

These records enable future users to reconstruct and verify the dataset.

Common Challenges

Researchers frequently encounter:

  • Missing metadata
  • Inconsistent annotations
  • Duplicate samples
  • Repository updates
  • Large file transfers
  • Incomplete downloads
  • Storage limitations

The CDI Data Acquisition System provides a structured framework for addressing these challenges.

By combining structured workflows, validation procedures, provenance tracking, and reproducible documentation, researchers can construct high-quality reference datasets suitable for downstream biological analysis.

Concluding Remarks

The CDI Data Acquisition System provides a reproducible framework for discovering, acquiring, validating, storing, and assembling public omics datasets.

Although the examples presented throughout this guide focus primarily on microbiome sequencing studies, the same principles can be applied across a broad range of omics domains including RNA-Seq, single-cell sequencing, genomics, and multi-omics integration.

The combination of systematic study discovery, metadata-driven sample selection, validated data acquisition, cloud-based storage, and reference dataset assembly creates a foundation for reproducible biological research and data-driven discovery.
